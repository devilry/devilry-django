GET /administrator/restfulsimplifiedrelatedstudent/id¶

Retrieve a related student.

The underlying data model where the item is retrieved from is defined in devilry.apps.core.models.relateduser.RelatedStudent. The Simplified API that the server forwards this request to is devilry.apps.administrator.simplified.SimplifiedRelatedStudent.read().

Request¶

Example¶

GET /administrator/restfulsimplifiedrelatedstudent/id

Required request parameters encoded in the URL¶

id¶

The unique identifier of a related student. You will typically get this id in response from a search.

Response¶

On success¶

Example¶

200 OK

{
    id: 15,
    username: "My example value",
    period: 15
}

Success response details¶

As long as the related student is deleted without an error, the response is HTTP 200 with the requested data in the body. The result always contains the following fields:

id
Actual location of the field:
devilry.apps.core.models.relateduser.RelatedStudent
About the field:
Autogenerated identifier.
Type
Integer
username
Actual location of the field:
devilry.apps.core.models.relateduser.RelatedStudent
About the field:
A username. Max 30 characters, a-z, A-Z, and _
Type
String
period
Actual location of the field:
devilry.apps.core.models.Period
About the field:
Autogenerated identifier.
Type
Integer

On error¶

On errors, we respond with one of the HTTP Error status codes.
